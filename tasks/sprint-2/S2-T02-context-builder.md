## Objective

Implement ContextBuilder that evaluates the brief, runs Interactive Mode (max 5 questions), and builds `context.md` for AI consumption. This component bridges the brief input and AI generation by enriching context through targeted questions.

---

## Scope

### Components to Create

1. **`src/core/ContextBuilder.ts`** — ContextBuilder service
   - `evaluateBrief(briefContent: string)` — analyze brief completeness
   - `runInteractiveMode(briefContent: string, answers?: Record<string, string>)` — ask up to 5 questions
   - `buildContext(briefContent: string, answers: Record<string, string>)` — generate context.md
   - `saveContext(projectPath: string, context: string)` — write to `input/context.md`

2. **`src/core/types.ts`** (extend if needed)
   - `BriefEvaluation` interface — completeness score, missing fields, suggested questions
   - `InteractiveQuestion` interface — question text, field name, required/optional

3. **Prompt Template**
   - `templates/prompts/brief-evaluation.prompt.md` — prompt for brief analysis
   - `templates/prompts/context-builder.prompt.md` — prompt for context generation

### Interactive Mode Logic

From SDD §6.4.1:
- Max 5 questions per brief
- Questions generated by AI based on brief gaps
- User answers stored in memory (not persisted to state yet)
- If `--skip-interactive` flag is used, skip questions and use brief as-is
- Questions should cover: project goals, stakeholders, constraints, timeline, success criteria

### Context.md Structure

From SDD §6.4.2:
```markdown
# Project Context

## Brief Summary
[AI-generated summary of brief]

## Project Information
- Project Name: [extracted]
- Client: [extracted]
- Timeline: [extracted]
- Budget: [if mentioned]

## Stakeholders
[extracted from brief + interactive answers]

## Goals & Objectives
[extracted from brief + interactive answers]

## Constraints & Requirements
[extracted from brief + interactive answers]

## Success Criteria
[extracted from brief + interactive answers]

## Additional Context
[AI-generated synthesis of all information]
```

## Acceptance Criteria

1. **Brief Evaluation**
   - Returns completeness score (0-10)
   - Identifies missing key fields
   - Generates targeted questions for gaps
   - Uses AIGateway for AI calls

2. **Interactive Mode**
   - Asks max 5 questions
   - Accepts user input via CLI
   - Supports `--skip-interactive` flag
   - Stores answers in memory object

3. **Context Building**
   - Generates structured context.md
   - Includes brief summary
   - Includes extracted project info
   - Includes AI-synthesized additional context
   - Uses AIGateway for AI calls

4. **Context Persistence**
   - Writes to `input/context.md`
   - Uses FileManager for atomic write
   - Creates `input/` directory if not exists

5. **Error Handling**
   - Handles empty brief gracefully
   - Handles AI errors with retry (via AIGateway)
   - Validates context.md before writing

6. **Unit Tests**
   - Test brief evaluation with mock AI
   - Test interactive mode with mock input
   - Test context building with mock AI
   - Test context persistence
   - Test `--skip-interactive` flag

## Implementation Notes

1. **AI Prompts**: Use prompt templates from `templates/prompts/` directory. Load via FileManager.

2. **CLI Interaction**: Use oclif's built-in prompt utilities (`ux.prompt`) for interactive questions.

3. **Brief Analysis**: The brief evaluation prompt should return JSON with:
   ```json
   {
     "completenessScore": 7,
     "missingFields": ["stakeholders", "timeline"],
     "questions": [
       {
         "field": "stakeholders",
         "question": "Who are the key stakeholders for this project?",
         "required": true
       }
     ]
   }
   ```

4. **Context Generation**: The context builder prompt should receive:
   - Brief content
   - Interactive answers (if any)
   - Output: structured markdown for context.md

5. **Skip Interactive**: When `--skip-interactive` is set, bypass question generation and use brief directly for context building.

6. **Error Recovery**: If AI fails during brief evaluation, default to asking generic questions (project name, goals, stakeholders, timeline, constraints).
